Functions and Object-Oriented Programming in JavaScript

This guide is intended to give you a basic understanding of functions and an introduction to Object-Oriented Programming (OOP) concepts. By the end of this lesson, you should feel confident about how to use functions effectively and why OOP can make your code more organised, maintainable, and scalable.

Organised

  • What it means: Your code is neat and structured, making it easy to understand.
  • Example: Imagine a bookshelf where books are sorted by category (fiction, history, etc.). It's easy to find a book when everything has a proper place.
  • In coding: Group related pieces of code together, name your functions and variables clearly, and avoid clutter.

Maintainable

  • What it means: Your code is easy to update, fix, or add new features without breaking everything.
  • Example: Think of a car that’s designed so mechanics can easily replace parts without disassembling the whole car.
  • In coding: Write clean and modular code (e.g., using functions or classes) so you can change one part without affecting the rest.

Scalable

  • What it means: Your code can handle growth—more users, more data, or more features—without major rewrites.
  • Example: A restaurant that can easily add more tables to serve more customers.
  • In coding: Use efficient methods and good design so your app runs smoothly even if it grows (e.g., handling 10 users vs. 1,000 users).

1. Understanding Functions

1.1 What is a Function?

A function is a reusable block of code that performs a specific task. Instead of writing the same code repeatedly, you can define it once in a function, then call (or “invoke”) the function whenever you need that task performed.

1.2 Why Use Functions?

  1. Reusability: You write the logic once and can call it multiple times.
  1. Maintainability: It’s easier to debug and modify code in a single place.
  1. Readability: Functions make your code more organised by grouping related logic together.
  1. Avoid Repetition: Repetition of logic can lead to mistakes and inconsistencies.

1.3 Basic Example

Imagine you want to calculate the average goals scored in a football match:

function calculateAverageGoals(totalGoals, matches) {
  return totalGoals / matches;
}

const avgGoals = calculateAverageGoals(30, 10);
console.log("Average goals scored per match:", avgGoals);

1.4 Functions with No Parameters

Functions don’t necessarily have to accept parameters. You can simply define a function to perform a specific task without needing inputs:

function cheerForTeam() {
  console.log("Go, Team, Go!");
}

cheerForTeam();

1.5 Arrow Functions - Optional Reading (ES6+ Syntax)

In modern JavaScript, you can also define functions using arrow function syntax:

const calculateAverageGoals = (totalGoals, matches) => {
  return totalGoals / matches;
};

// Or even shorter if there's only one statement in the function body:
const calculateAverageGoalsShort = (totalGoals, matches) => totalGoals / matches;

3. When Functions Become Numerous: Considering OOP

As you can see from the examples and exercises above, we have a growing list of functions: registering teams, recording results, calculating stats, displaying standings, etc. Over time, your application might grow even further, adding more functionalities. Soon, keeping track of how all these functions relate to each other can become confusing.

This is where Object-Oriented Programming (OOP) can help.

3.1 What is OOP?

Object-Oriented Programming is a programming paradigm where we structure our code around objects. Each object can contain:

  • Properties (like variables or data)
  • Methods (like functions, but tied to the object)

3.2 Benefits of OOP

  1. Encapsulation: Group related data and methods together in a single unit (object).
  1. Reusability (via Inheritance): One class can inherit properties and methods from another.
  1. Modularity: Organize code into logical sections or “blueprints,” making it easy to navigate and maintain.
  1. Maintainability: Changes in one object/class often don’t affect the code in other objects/classes as long as the interface remains the same.

3.3 What Are Classes?

In Object-Oriented Programming, a class is like a blueprint for creating objects. It defines what properties (data) and methods (functions) the objects will have. While objects are the actual instances you interact with in your program, the class serves as the template to create them.

3.3.1 Class Syntax

In JavaScript, you define a class using the class keyword. Here’s a basic example:

class Team {
  // The blueprint for a team
  constructor(name, city) {
    this.name = name; // Property
    this.city = city; // Property
  }

  // Method
  displayInfo() {
    console.log(`Team: ${this.name}, City: ${this.city}`);
  }
}

// Creating an object (instance) from the class
const team1 = new Team("Dream FC", "Dreamland");
team1.displayInfo(); // Output: Team: Dream FC, City: Dreamland

3.3.2 What Is a Constructor?

The constructor is a special method in a class that is automatically called when you create a new object (or instance) of that class. It is used to initialize the object’s properties.

  • Purpose: The constructor is like a setup function that runs once when an object is created.
  • Syntax: The constructor method is defined using the keyword constructor.

Here’s an example:

class Player {
  constructor(name, position, age) {
    this.name = name; // Assign the name to the object
    this.position = position; // Assign the position to the object
    this.age = age; // Assign the age to the object
  }

  // Method to display player info
  displayInfo() {
    console.log(`${this.name} plays as a ${this.position} and is ${this.age} years old.`);
  }
}

const player1 = new Player("John Doe", "Forward", 28);
player1.displayInfo(); // Output: John Doe plays as a Forward and is 28 years old.

Key Points About the Constructor:

  1. The constructor is optional. If you don’t need to initialize any properties during object creation, you can omit it.
  1. A class can have only one constructor. If you define more than one, you’ll get an error.
  1. You can pass arguments to the constructor to initialize an object with dynamic values.

3.3.4 Adding Methods to Classes

Methods are functions defined inside a class that operate on the object’s data (properties). These methods can manipulate or display the object’s data.

Example:

class Player {
  constructor(name, position, goals) {
    this.name = name;
    this.position = position;
    this.goals = goals;
  }

  // Method to update the number of goals
  scoreGoal() {
    this.goals += 1;
    console.log(`${this.name} has scored! Total goals: ${this.goals}`);
  }

  // Method to display player stats
  displayStats() {
    console.log(`Player: ${this.name}, Position: ${this.position}, Goals: ${this.goals}`);
  }
}

const player1 = new Player("John Doe", "Forward", 15);
player1.scoreGoal(); // John Doe has scored! Total goals: 16
player1.displayStats(); // Player: John Doe, Position: Forward, Goals: 16

3.3.5 Why Use Classes Instead of Functions?

In the exercises earlier, we used functions to manage tasks like registering teams, recording results, and calculating stats. As the complexity increased, these functions became harder to manage. Using classes instead of standalone functions offers several advantages:

  1. Grouping Related Data and Behavior:
    • A class ties data (properties) and functions (methods) together, making your code more organized.
    • Example: A Player class contains all data and behavior related to a player, instead of having separate functions like addPlayer() or updateGoals() scattered throughout the code.
  1. Reusability:
    • A class can be reused to create multiple objects with the same structure but different data.
    • Example: You can create multiple teams from a single Team class, each with unique names, cities, and players.
  1. Readability and Maintainability:
    • Classes make it easier to understand how data and behavior are connected.
    • If something needs to be changed, you can update the class definition instead of searching for every instance where related functions are used.
  1. Extensibility:
    • Classes can be extended using inheritance, which allows new classes to reuse and add to the functionality of existing ones. (This will be covered later.)

Consider the earlier example where we registered teams, recorded match results, and calculated statistics. With classes, we can group all the related properties and methods into logical units.

4. Transitioning the Football Example to OOP

Instead of having global variables like teams and multiple standalone functions, we can create classes (blueprints for objects) that logically organize our data and methods.

4.1 Creating a Team Class

class Team {
  constructor(name) {
    this.name = name;
    this.wins = 0;
    this.draws = 0;
    this.losses = 0;
    this.goalsScored = 0;
    this.goalsConceded = 0;
  }

  // Method to update stats after a match
  recordMatch(goalsFor, goalsAgainst) {
    this.goalsScored += goalsFor;
    this.goalsConceded += goalsAgainst;

    if (goalsFor > goalsAgainst) {
      this.wins += 1;
    } else if (goalsFor < goalsAgainst) {
      this.losses += 1;
    } else {
      this.draws += 1;
    }
  }

  // Method to display team info
  getTeamInfo() {
    return `Team: ${this.name}, Wins: ${this.wins}, Draws: ${this.draws}, Losses: ${this.losses}`;
  }
}

Here, each Team object has its own data (name, wins, draws, etc.) and methods (recordMatch, getTeamInfo). This is encapsulation in action: the data and the logic to handle that data are combined in one place.

4.2 Creating a League Class

We might have a separate class League to keep track of all teams and handle league-wide statistics.

class League {
  constructor() {
    this.teams = [];
  }

  addTeam(teamName) {
    const newTeam = new Team(teamName);
    this.teams.push(newTeam);
  }

  recordMatchResult(homeTeamName, awayTeamName, homeGoals, awayGoals) {
    const homeTeam = this.teams.find(team => team.name === homeTeamName);
    const awayTeam = this.teams.find(team => team.name === awayTeamName);

    if (!homeTeam || !awayTeam) {
      console.log("Error: One or both teams not found in the league!");
      return;
    }

    homeTeam.recordMatch(homeGoals, awayGoals);
    awayTeam.recordMatch(awayGoals, homeGoals);
  }

  calculateLeagueStats() {
    let totalTeamMatches = 0;
    let totalGoals = 0;
    this.teams.forEach(team => {
      totalTeamMatches += (team.wins + team.draws + team.losses);
      totalGoals += team.goalsScored;
    });

    const totalMatches = totalTeamMatches / 2;
    const averageGoalsPerMatch = totalMatches ? (totalGoals / totalMatches) : 0;

    return {
      totalMatches,
      totalGoals,
      averageGoalsPerMatch
    };
  }

  displayStandings() {
    console.log("League Standings:");
    this.teams.forEach(team => {
      console.log(team.getTeamInfo());
    });
  }
}

Now we have a clear separation:

  • Team: The data and methods for a single team.
  • League: Manages multiple teams and coordinates matches, stats, and standings.

4.3 Using Our Classes

// Creating a league instance
const myLeague = new League();

// Add teams
myLeague.addTeam("Dream FC");
myLeague.addTeam("Phoenix United");
myLeague.addTeam("Sunrise FC");

// Record matches
myLeague.recordMatchResult("Dream FC", "Phoenix United", 3, 1);
myLeague.recordMatchResult("Dream FC", "Sunrise FC", 2, 2);
myLeague.recordMatchResult("Phoenix United", "Sunrise FC", 4, 3);

// Display standings
myLeague.displayStandings();

// Calculate stats
const leagueStats = myLeague.calculateLeagueStats();
console.log("League Stats:", leagueStats);

5. OOP Principles Recap

Here are some key OOP principles demonstrated:

  1. Encapsulation: We contained the Team data (wins, draws, losses, etc.) and the behavior (recordMatch, getTeamInfo) in the Team class. We contained the league-wide logic in the League class.
  1. Abstraction: We’re hiding internal details of how exactly the match results get recorded. From outside the Team class, you just call team.recordMatch(...) without caring about the exact steps involved.
  1. Inheritance (Not explicitly shown yet): One class can extend another class to inherit its properties and methods. For instance, if you had a specialized type of team (like a “Veteran Team” with some special properties), you could extend the Team class.
  1. Polymorphism (Not explicitly shown yet): Different classes can have methods with the same name but different implementations. This is often used in more complex situations.

  1. Adding a Tournament or Cup
    • Create a new class Cup that also has teams but eliminates teams when they lose.
    • This will help you practice OOP by creating a different object with its own logic.
  1. Enhancing the Team Class
    • Add a property for the name of the team’s coach.
    • Add a method to change the coach mid-season (changeCoach()).
    • Print out the updated coach information in getTeamInfo().
  1. Explore Inheritance
    • Create a SpecialTeam class that extends Team. For example, a “ChampionTeam” might start the season with some “bonus points” or some special rules.
  1. Stats Output
    • Write a method in League to display which team has the highest goals scored, which team has the highest goals conceded, etc. This encourages reading and processing your array of Team objects more deeply.
  1. Practice with Another Scenario
    • If you’re interested in other examples, pick another scenario (like a “Movie Collection” or “Online Store Inventory System”) to build out classes and methods.
    • The logic is the same: group related data (properties) and behaviors (methods) into classes that make sense for the domain you’re modeling.

7. Additional Interesting Examples Outside Football

While the football example helps you grasp the concepts, it’s beneficial to see OOP can be applied to almost any domain:

  1. Movie Library
    • Classes: Movie, Actor, MovieCollection
    • Properties: Title, genre, releaseYear, rating, etc.
    • Methods: addMovie(), removeMovie(), searchByActor(), getHighestRatedMovie()
  1. Online Store
    • Classes: Product, Cart, Customer, Order
    • Properties: Product.name, Cart.items, Customer.name, etc.
    • Methods: addToCart(), calculateTotal(), placeOrder()

The idea remains the same—create classes to represent entities in your problem, with properties that describe the entity and methods that perform actions on it.
